# دليل إعداد قاعدة البيانات المستدامة على Render

## المشكلة
عند استخدام SQLite مع ملف محلي في بيئة مثل Render، يتم حذف قاعدة البيانات مع كل Deploy جديد لأن الحاوية تُعاد إنشاؤها من الصفر.

## الحل: استخدام قاعدة بيانات سحابية مستدامة

### الخيار 1: PostgreSQL على Render (موصى به)

#### الخطوات:

1. **إنشاء قاعدة بيانات PostgreSQL على Render:**
   - اذهب إلى لوحة تحكم Render
   - انقر على "New" → "PostgreSQL"
   - اختر الخطة المجانية (Free)
   - انتظر حتى يتم إنشاء قاعدة البيانات

2. **الحصول على رابط الاتصال:**
   - بعد إنشاء قاعدة البيانات، انسخ "Internal Database URL"
   - سيكون بالشكل: `postgresql://user:password@hostname:5432/dbname`

3. **إضافة متغير البيئة على Render:**
   - اذهب إلى خدمة الويب الخاصة بك على Render
   - انقر على "Environment"
   - أضف متغير جديد:
     - الاسم: `DATABASE_URL`
     - القيمة: رابط قاعدة البيانات الذي نسخته

4. **إعادة تشغيل الخدمة:**
   - سيعيد Render تشغيل الخدمة تلقائياً
   - سيتم إنشاء الجداول تلقائياً عند التشغيل الأول

#### مميزات هذا الحل:
- ✅ البيانات مستدامة ولا تحذف مع الـ Deploy
- ✅ أداء أفضل من SQLite
- ✅ مناسب للتطبيقات الإنتاجية
- ✅ نسخ احتياطي تلقائي

---

### الخيار 2: استخدام SQLite مع تخزين خارجي (أقل توصية)

إذا كنت تريد الاستمرار مع SQLite، يمكنك استخدام تخزين خارجي مثل:

#### أ) Render Disk (Persistent Disk):
1. اذهب إلى خدمتك على Render
2. انقر على "Disks" → "Add Disk"
3. اختر حجماً مناسباً (مثلاً 1GB)
4. اضبط مسار التثبيت: `/app/data`
5. عدّل الكود ليستخدم: `DATABASE_URL=sqlite:////app/data/inspection.db`

**ملاحظة:** Render لا يدعم Persistent Disks في الخطة المجانية حالياً.

#### ب) تخزين سحابي خارجي:
- استخدم خدمات مثل AWS S3 مع SQLite عبر أدوات خاصة
- أو استخدم Google Cloud Storage
- هذا الحل معقد وغير موصى به للبداية

---

## التنفيذ الحالي في الكود

تم تعديل `database.py` ليدعم كلا الخيارين:

```python
# يقرأ من متغير البيئة DATABASE_URL
DB_PATH = os.environ.get("DATABASE_URL", "sqlite:///inspection.db")

# يدعم:
# - SQLite: sqlite:///path/to/file.db
# - PostgreSQL: postgresql://user:pass@host/db
```

## خطوات النشر على Render

### للقاعدة البيانات PostgreSQL:

1. أنشئ قاعدة بيانات PostgreSQL جديدة على Render
2. انسخ "Internal Database URL"
3. في خدمة الويب:
   - Environment → Add Variable
   - الاسم: `DATABASE_URL`
   - القيمة: `postgresql://...` (الرابط الداخلي)
4. أعد تشغيل الخدمة

### للتحقق من نجاح الإعداد:

1. افتح تطبيقك
2. أضف بعض البيانات (مستخدمين، جلسات، تقارير)
3. قم بعمل Deploy جديد
4. تحقق من أن البيانات ما زالت موجودة

## نصائح إضافية

### الأمان:
- ⚠️ لا تشارك رابط قاعدة البيانات أبداً
- ⚠️ استخدم المتغيرات البيئية دائماً
- ⚠️ لا تضع روابط قواعد البيانات في GitHub

### النسخ الاحتياطي:
- PostgreSQL على Render يقوم بنسخ احتياطي تلقائي
- يمكنك تصدير البيانات يدوياً من لوحة التحكم

### الترقية:
- ابدأ بالخطة المجانية
- عندما يزيد الاستخدام،رقّي إلى خطة أعلى
- PostgreSQL يمكن ترقيته بدون فقدان البيانات

## استكشاف الأخطاء

### المشكلة: خطأ في الاتصال بقاعدة البيانات
**الحل:** تأكد من أن `DATABASE_URL` صحيح وفي متغيرات البيئة

### المشكلة: الجداول غير موجودة
**الحل:** التطبيق ينشئ الجداول تلقائياً عند أول تشغيل، تحقق من logs

### المشكلة: البيانات تُحذف بعد الـ Deploy
**الحل:** تأكد من استخدام قاعدة بيانات خارجية وليس ملف محلي
