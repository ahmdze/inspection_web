# 📋 خطوات حل مشكلة حذف قاعدة البيانات على Render

## ✅ ما تم تعديله في الكود

### 1. **تعديل `database.py`**
- أصبح يدعم قراءة مسار قاعدة البيانات من متغير البيئة `DATABASE_URL`
- يدعم SQLite المحلي (للتطوير)
- يدعم PostgreSQL السحابي (للإنتاج على Render)
- يتعرف تلقائياً على نوع قاعدة البيانات من الرابط

### 2. **تعديل `.gitignore`**
- تمت إضافة `*.db` لمنع رفع ملفات قاعدة البيانات إلى GitHub
- تمت إضافة `reports/` لمنع رفع التقارير المؤقتة
- تمت إضافة `.env` لحماية المتغيرات الحساسة

### 3. **تعديل `Dockerfile`**
- تمت إضافة `VOLUME ["/app/data"]` لإنشاء مساحة تخزين مستدامة
- تمت إضافة `RUN mkdir -p /app/data` لإنشاء مجلد البيانات

### 4. **تعديل `requirements.txt`**
- تمت إضافة `psycopg2-binary` لدعم PostgreSQL

## 🚀 خطوات التطبيق على Render

### الخطوة 1: إنشاء قاعدة بيانات PostgreSQL على Render

1. سجّل الدخول إلى [لوحة تحكم Render](https://dashboard.render.com/)
2. انقر على **"New"** → **"PostgreSQL"**
3. اختر الإعدادات:
   - **Name**: اسم مناسب مثل `inspection-db`
   - **Region**: نفس منطقة خدمة الويب
   - **Database**: `inspection_db`
   - **Plan**: اختر **Free** (مجاني)
4. انقر على **"Add Database"**
5. انتظر حتى تكتمل عملية الإنشاء (قد تستغرق دقيقة أو دقيقتين)

### الخطوة 2: الحصول على رابط الاتصال

1. بعد إنشاء قاعدة البيانات، افتحها من لوحة التحكم
2. ابحث عن **"Internal Database URL"**
3. انسخ الرابط، سيكون بالشكل:
   ```
   postgresql://user:password@hostname:5432/dbname?sslmode=require
   ```

### الخطوة 3: إضافة متغير البيئة على Render

1. اذهب إلى **خدمة الويب** الخاصة بك على Render
2. انقر على تبويب **"Environment"**
3. انقر على **"Add Variable"**
4. أضف المتغير التالي:
   - **Key**: `DATABASE_URL`
   - **Value**: الصق رابط قاعدة البيانات الذي نسخته
5. انقر على **"Save Changes"**

### الخطوة 4: دفع التعديلات إلى GitHub

```bash
cd /workspace
git add .
git commit -m "Fix: Add database persistence support for Render"
git push origin qwen-code-10dbaafb-d535-4d70-8f40-bb0d183d3dbd
```

**ملاحظة:** Render سيقوم تلقائياً بإعادة النشر عند الدفع إلى GitHub.

### الخطوة 5: التحقق من نجاح الإعداد

1. انتظر حتى يكتمل الـ Deploy على Render
2. افتح تطبيقك
3. سجّل الدخول (المستخدم الافتراضي: `admin` / كلمة المرور: `123`)
4. أضف بعض البيانات التجريبية:
   - أضف مستخدم جديد
   - أنشئ جلسة تفتيش
   - أضف تقرير
5. قم بعمل Deploy جديد (يمكنك إضافة أي تغيير بسيط)
6. بعد اكتمال الـ Deploy، تحقق من أن البيانات ما زالت موجودة!

## 🔍 استكشاف الأخطاء

### المشكلة: خطأ في الاتصال بقاعدة البيانات

**الأعراض:** التطبيق لا يعمل أو يظهر خطأ اتصال

**الحلول:**
1. تأكد من أن `DATABASE_URL` صحيح في Environment Variables
2. تحقق من أن قاعدة البيانات PostgreSQL تعمل على Render
3. راجع سجلات التطبيق (Logs) على Render لمعرفة التفاصيل

### المشكلة: الجداول غير موجودة

**الأعراض:** خطأ في قاعدة البيانات أو جداول فارغة

**الحل:** 
- التطبيق ينشئ الجداول تلقائياً عند أول تشغيل
- إذا لم يتم الإنشاء، راجع Logs لمعرفة السبب

### المشكلة: البيانات تُحذف بعد الـ Deploy

**الحل:**
- تأكد من استخدام `DATABASE_URL` مع PostgreSQL وليس ملف محلي
- تحقق من أن المتغير موجود في Environment Variables على Render
- تأكد من عدم وجود ملف `inspection.db` في الكود على GitHub

## 💡 نصائح مهمة

### الأمان
- ⚠️ **لا تشارك** رابط قاعدة البيانات أبداً
- ⚠️ **لا تضع** روابط قواعد البيانات في GitHub
- ✅ استخدم دائماً **Environment Variables** للمتغيرات الحساسة

### النسخ الاحتياطي
- Render يقوم بعمل نسخ احتياطي تلقائي لـ PostgreSQL
- يمكنك تصدير البيانات يدوياً من لوحة تحكم Render
- يُنصح بعمل Export دوري للبيانات المهمة

### التكلفة
- خطة PostgreSQL المجانية على Render توفر:
  - 1GB تخزين
  - قاعدة بيانات مشتركة
  - مناسبة للتطبيقات الصغيرة والمتوسطة
- عند زيادة الاستخدام، يمكنك الترقية لخطة أعلى

## 🎯 الخلاصة

**المشكلة:** كانت قاعدة البيانات SQLite تُحذف مع كل Deploy لأن الملف كان جزءاً من الحاوية المؤقتة.

**الحل:** استخدام PostgreSQL السحابي على Render مع متغير بيئة `DATABASE_URL`.

**النتيجة:** البيانات ستبقى دائمة ولا تُحذف مع الـ Deploys الجديدة! 🎉
