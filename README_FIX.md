# إصلاح صفحة باني النموذج (Form Builder)

## المشكلة
كانت صفحة `/admin/form-builder` تظهر خطأ "Internal Server Error" بسبب عدم تطابق اسم العمود في قاعدة البيانات مع الكود.

## الحل المطبق

### 1. تم إصلاح اسم العمود في `database.py`
- تغير الاسم من `Recommendation_Categories` (بأحرف كبيرة) إلى `recommendation_categories` (بأحرف صغيرة)
- هذا يتوافق مع معايير تسمية SQLAlchemy ومتغيرات Python

### 2. تم تحديث `main.py` لمعالجة فئات التوصيات
- إضافة كود لقراءة قيم `rec_cat` من النموذج
- حفظها كـ JSON في عمود `recommendation_categories`

### 3. تم إضافة ترحيل تلقائي لقاعدة البيانات
- عند بدء التطبيق، يتم التحقق من وجود العمود `recommendation_categories`
- إذا لم يكن موجوداً، يتم إضافته تلقائياً
- يعمل الترحيل مع SQLite و PostgreSQL (قاعدة بيانات Render)

## الخطوات المطلوبة على Render

### الطريقة 1: إعادة تشغيل التطبيق (الأسهل)
1. اذهب إلى لوحة تحكم Render
2. اختر التطبيق الخاص بك
3. انقر على "Manual Deploy" أو "Restart"
4. عند إعادة التشغيل، سيتم تطبيق الترحيل تلقائياً

### الطريقة 2: تشغيل سكريبت الترحيل يدوياً
إذا استمرت المشكلة، يمكنك تشغيل السكريبت المرفق `migration_fix.py` عبر وحدة تحكم Render:

```bash
python migration_fix.py
```

## الملفات المعدلة
- `database.py`: إصلاح اسم العمود وإضافة ترحيل تلقائي
- `main.py`: إضافة معالجة فئات التوصيات عند حفظ الحقول

## ملاحظة مهمة
التطبيق مصمم للعمل مع متغير البيئة `DATABASE_URL` الذي توفره Render تلقائياً عند ربط قاعدة بيانات PostgreSQL.
